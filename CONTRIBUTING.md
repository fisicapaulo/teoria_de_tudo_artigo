# Contribuindo

Obrigado por considerar contribuir!

- Abra uma issue descrevendo o problema ou proposta.
- Para PRs:
  - Mantenha commits atômicos e mensagens claras.
  - Adicione/atualize testes.
  - Rode lint e testes localmente antes do PR.

Checklist:
- [ ] Código com docstrings e tipagem quando aplicável.
- [ ] Nenhum `DeprecationWarning` nos logs.
- [ ] Atualização de README/artefatos quando necessário.
